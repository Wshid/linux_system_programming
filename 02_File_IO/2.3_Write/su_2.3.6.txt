write() 동작방식

호출 반환시 사용자 영역에서 커널이 넘긴 버퍼에서 커널 버퍼로 데이터가 복사는 됨
    -> 하지만 의도한 목적지에 데이터를 썼다는 보장을 못함
    
단, 사용자 영역 app이 write() 시스템콜 호출시, 커널은 몇가지 점검 후 데이터를 버퍼로 복사함

변경된 버퍼를 수집하여 최적 수준으로 정렬한 다음에 디스크에 쓴다(writeback ; 쓰기저장 이라고 함)

또한, 여유가 생길때까지 쓰기 작업을 늦춰서 한꺼번에 여러작업을 모아 배치로 수행한다.
    -> 하지만 이렇다고 해서 좀전에 작성한 데이터를 불러올때는 버퍼에서 불러옴

쓰기 순서 강제 : 데이터베이스 에서만, 나머지는 적용되지 않음

커널은 지연된 스기에 대한 위험 최소화를 위해 최대 버퍼나이를 만들어서
나이가 꽉찬 변경된 버퍼를 빠짐없이 기록함
    /proc/sys/vm/dirty_expire_centiseconds 에서 1/100초단위로 설정이 가능
    
파일 버퍼의 쓰기 저장 기능을 강제로 켜거나, 모든 쓰기를 동기식으로 만들 수 있음
