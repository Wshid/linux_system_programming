벡터 입출력이란?
    한번의 시스템 콜을 사용해서 여러 개의 버퍼 벡터에 쓰거나
    여러개의 버퍼 벡터로 읽어들일 때 사용하는 입출력 메서드
    
    데이터를 여러 버퍼 벡터로 흩뿌리거나, 여러 버퍼 벡터로 모으기 때문에 이름이 붙여짐
    
    일반적인(2장)에서 다루었던 stdio의 읽기와 쓰기는 선형 입출력이라고 함

장점
    1. 데이터가 분리 되어 있을때 직관적인 조작 가능 // 미리 정의된 구조체의 여러 필드에 걸쳐 데이터 분산시
    2. 하나의 벡터 입출력 연산은 여러번의 선형 입출력 연산 대체 가능
    3. sys.call의 호출 횟수 감소, 선형 입출력 보다 최적화된 구현 제시
    4. 벡터 입출력 연산 중에 다른 ps가 끼어들 수 없음 // 원자성
