event_ctl()로 전달하는
    event.events=EPOLLET // fd 이벤트 모니터가 레벨 트리거가 아닌 에지트리거로 동작
    
파이프 연산
    1. 출력하는 측에서 파이프에 1KB 데이터를 쓴다
    2. 입력받는 쪽에서 파이프에 대해 epoll_wait() 수행후
        파이프에 데이터가 들어와서 읽을 수 있는 상태가 되기를 기다린다.

레벨 트리거로 이벤트 모니터링시, 2단계의 epoll_wait은 즉시 반환, 파이프 읽을 준비가 되었음을 알려줌
에지 트리거로 모니터 설정시, 1단계가 완료될때까지 호출이 반환되지 않음
    -> epoll_wait()호출하는 시점에 파이프를 읽을 시점이 되더라도
        파이프에 데이터가 들어오기 전까지는 반환하지 않음

기본 동작 방식은 레벨 트리거
    poll, select도 레벨트리거
    
에지 트리거는 일반적으로 논블록킹 입출력 화룡ㅇ시 사용,
    EAGAIN을 주의깊게 활용해야 함



레벨 트리거 인터럽트 신호 = 특정 레벨을 유지할때 발생
    -> 이벤트의 상태(신호의 레벨)을 목적으로 할때 사용
에지 트리거 인터럽트 신호 = 신호가 하강하거나 상승하는 동안 발생
    -> 이벤트 자체(신호가 움직이는)을 목적으로 할때 사용
    
읽어들이려는 fd가 있다 할때
    epoll을 레벨트리거를 사용하면
        fd가 읽기 가능한 상태가 되었을때 알림을 받게 됨
            -> 상태의 지속이 알림을 발생
    에지트리거 사용시
        읽을 수 있는 첫번째 데이터가 들어오면 알림을 받게됨
            알림이 발생되는 것은 에지, 상태의 변화임
