최신 시스템에서
    디스크와 시스템 나머지 부분간의 성능차이가 매우 큼

따라서 최신 OS의 커널은 IO Scheduler를 구현하여, 입출력 요청의 순서와 시간을 조작,
    디스크 탐색 횟수를 최소화 한다.

디스크 접근에 따른 성능 하락을 줄이기 위해 바쁘게 동작함

4.6.1=== 디스크 주소 지정 방식
hdd=실린더, 헤드, 섹터 | CHS 주소 지정 방식 이용

hdd = Platter * n
Plateer=디스크, 스핀들, 읽기/쓰기 헤더

플레터를 CD로 생각, 여러개의 플레터는 겹쳐진 CD라고 생각
    각각의 플래터는 CD처럼 원형의 트랙으로 나누어짐
        트랙들은 정수개의 섹터로 나누어짐

데이터 저장 디스크 위치 => 실린더, 헤드, 섹터 값을 필요로 함

실린더 - 데이터가 위치한 트랙

여러 플래터가 있다면, 실린더는 일종의 원기둥을 형성
    : 각 디스크의 중심에서 같은 거리만큼 떨어져 있는 트랙

헤드 값 : 요청한 읽기/쓰기 헤드(정확한 플래터)의 정확한 값 구분
섹터 값 : 트랙에 위치한 정확한 섹터 찾기
    -> 탐색 완료

최신 하드디스크는 컴퓨터가 실린더, 헤드, 섹터 값으로 디스크에서 데이터를 읽도록 강제 x
    -> 대신, 블록번호(물리 블록 | 디바이스 블록)로 특정 섹터에 대응되도록 함

논리블록주소지정(Logical Block Addressing) : 블록 번호를 이용하여 hdd의 주소 지정

내부적으로 이 블록 번호 가지고 CHS 번호로 변환
    대부분 논리블록 n과 n+1은 물리적으로 인접하는 경우가 있음
    
fs는 소프트웨어로만 존재
    fs는 논리블록이라고하는 독자적인 단위를 이용하여 사용
    논리블록=물리블록의 정수배

fs의 논리블록은
    하나 이상의 디스크의 물리블록에 맵핑됨
