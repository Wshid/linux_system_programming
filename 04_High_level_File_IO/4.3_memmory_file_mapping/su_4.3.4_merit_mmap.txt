read()/write()보다 mmap()이용해서 파일 조작시

- read()/write()떄의 불필요한 복사를 방지함 // 사용자 지역의 버퍼
- 다른 시스템 콜 호출이나 Context Swtiching(문맥교환) 오버헤드 발생 x
    -> 메모리에 접근하는 것처럼 단순 동작
    (페이지 폴트 가능성 제외)
- 여러 ps가 같은 객체 메모리 맵핑시, 데이터는 모든 ps 사이 공유됨
    -> 읽기 전용이나, MAP_SHARED로 쓰기 가능시, 전체 공유됨
    -> MAP_PRIVATE 모드 맵핑시 copy-on-write 전까지만 페이지 공유
- lseek() 사용 안해도 간단한 포인터 조작으로 맵핑 영역 탐색이 가능
